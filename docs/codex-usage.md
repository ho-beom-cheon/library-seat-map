# Codex 사용 가이드

이 문서는 `library-seat-map`에서 Codex에게 작업을 요청할 때 사용할 기준을 정리합니다.

## 기본 요청 형식

```text
AGENTS.md 기준으로 진행해줘.

이번 작업은 <작업 요약>이야.

목표:
-

진행 방식:
- 관련 이슈가 없으면 먼저 이슈 초안을 제안
- 전용 브랜치 생성
- 요청 범위 파일만 수정
- 가능한 검증 수행
- 커밋 메시지, PR 제목/본문, 릴리스 노트 후보, 다음 작업 브랜치명 제안
```

## 구현 작업 요청 시 포함할 내용

- 관련 이슈 번호 또는 이슈 초안 요청
- 작업 대상 기능 또는 문서
- 포함할 범위와 제외할 범위
- 기대 동작
- 실행할 수 있는 검증 명령
- 민감정보나 공개하면 안 되는 정보의 범위

## 프로젝트 특화 주의사항

- 공공데이터포털 서비스키는 서버 환경변수로만 관리합니다.
- 프론트엔드에는 공공데이터 서비스키를 두지 않습니다.
- 사용자 현재 위치는 저장하지 않습니다.
- 좌석 상태는 "최근 갱신 기준"으로 표현합니다.
- 좌석 보장, 예약 가능, 무조건 이용 가능 같은 표현은 사용하지 않습니다.

## 완료 보고에 포함할 내용

- 작업 요약
- 관련 이슈
- 변경 파일
- 검증 결과
- 커밋 메시지 제안
- PR 제목과 본문 제안
- 릴리스 노트 후보
- 다음 작업 브랜치 제안
